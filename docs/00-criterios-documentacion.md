# Criterios de documentación

## Objetivo

Definir las normas y principios para mantener una documentación técnica clara, consistente y reutilizable.

## Principios generales

- Documentar únicamente lo que existe o se ha probado
- Priorizar claridad sobre cantidad
- Evitar duplicidad de información
- Mantener un enfoque reutilizable (no dependiente de un proyecto concreto)

## Estructura

Cada documento debe incluir, cuando aplique:

### 1. Qué

Descripción breve de lo que se está haciendo.

### 2. Cómo

Pasos necesarios para llevarlo a cabo.

### 3. Por qué

Motivo técnico de la decisión o procedimiento.

### 4. Verificación

Cómo comprobar que funciona correctamente.

### 5. Problemas comunes

Errores típicos y cómo resolverlos.

## Organización

- Usar nombres de archivos numerados para mantener el orden
- Separar los contenidos en archivos pequeños y específicos
- Evitar archivos excesivamente largos

## Estilo

- Usar títulos claros (`##`, `###`)
- Evitar texto innecesario
- Incluir comandos cuando sea relevante
- Explicar los conceptos clave cuando aporten valor

## Mantenimiento

- Actualizar la documentación junto con los cambios del proyecto
- No dejar contenido obsoleto
- Revisar coherencia entre secciones
